# Схема ролей и прав

## 0) Основные принципы

1. Пользователь может иметь несколько ролей одновременно.  
   Роли описывают навыки/допуски, а не должность.

2. Права определяются по объединению ролей.  
   Если хотя бы одна роль даёт доступ — доступ есть.

3. Полная видимость всех мероприятий есть у ролей:
   - Менеджер
   - Старший инженер
   - Кладовщик

4. Управление жизненным циклом мероприятия — только Менеджер:
   - создание
   - редактирование
   - статусы
   - удаление
   - корзина

5. Старший инженер управляет комплектацией мероприятия:
   - собственное оборудование
   - арендованное оборудование

6. Кладовщик:
   - видит все мероприятия
   - не управляет мероприятиями
   - управляет только складом и состоянием оборудования

- Менеджер — владелец мероприятия
